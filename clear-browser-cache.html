🔧 브라우저 캐시 초기화 가이드

⚠️ 문제 상황: Next.js 정적 파일 (CSS, JS)이 404 오류로 로드되지 않는 경우

방법 1: 강력 새로고침 (권장)

Mac:

  1. Chrome/Safari 브라우저에서 Cmd + Shift + R 누르기
  2. 또는 개발자 도구 열고 새로고침 버튼 우클릭 → "강력 새로고침 및 캐시 비우기" 선택

Windows:

  1. Chrome/Edge 브라우저에서 Ctrl + Shift + R 또는 Ctrl + F5 누르기

방법 2: 개발자 도구에서 캐시 삭제

  1. 개발자 도구 열기 (F12 또는 Cmd/Ctrl + Shift + I)
  2. Application 탭 → Storage 섹션
  3. "Clear site data" 버튼 클릭
  4. 또는 Console에서 아래 명령 실행:
// Service Worker 제거 navigator.serviceWorker.getRegistrations().then(function(registrations) { for(let registration of registrations) { registration.unregister(); } }); // 모든 캐시 삭제 caches.keys().then(function(names) { for (let name of names) { caches.delete(name); } }); // 로컬 스토리지 초기화 localStorage.clear(); sessionStorage.clear();

방법 3: Chrome 설정에서 캐시 삭제

  1. Chrome 설정 → 개인정보 및 보안 → 인터넷 사용 기록 삭제
  2. "캐시된 이미지 및 파일" 체크
  3. 시간 범위: "전체 기간" 선택
  4. "인터넷 사용 기록 삭제" 클릭

방법 4: 시크릿/프라이빗 모드 사용

캐시 없이 깨끗한 상태로 테스트:

  • Chrome: Cmd/Ctrl + Shift + N
  • Safari: Cmd + Shift + N
  • Firefox: Cmd/Ctrl + Shift + P

서버 측 해결 방법 (이미 완료됨)

# Next.js 캐시 삭제 및 서버 재시작 rm -rf .next npm run dev
로그인 페이지로 이동 홈페이지로 이동
💡 팁: 개발 중에는 개발자 도구를 열어두고 Network 탭에서 "Disable cache" 체크박스를 활성화하면 캐시 문제를 방지할 수 있습니다.
